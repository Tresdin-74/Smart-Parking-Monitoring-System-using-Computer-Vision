# سیستم نظارت تصویری هوشمند پارکینگ (Smart Parking Surveillance System)

این پروژه یک سیستم کامل نظارت بر پارکینگ است که **بدون استفاده از هیچ مدل
یادگیری عمیق**، فقط با ابزارهای کلاسیک OpenCV پیاده‌سازی شده است.

## ساختار پروژه

```
exercise04_parking_system/
├── main.py            # فایل اصلی اجرای برنامه (حلقه پردازش ویدئو + داشبورد)
├── detector.py        # تشخیص اشغال/خالی بودن هر جای پارک
├── tracker.py         # تشخیص ورود/خروج خودرو و جهت حرکت
├── logger.py          # ثبت رویدادها در فایل‌های CSV با timestamp
├── utils.py           # توابع کمکی (CLAHE، مقایسه هیستوگرام، زمان)
├── spot_selector.py   # ابزار تعیین محل جاهای پارک با ماوس
├── parking_spots.json # خروجی spot_selector.py (مختصات جاهای پارک)
├── README.md          # همین فایل
└── report.md          # گزارش چالش‌ها و راه‌حل‌ها
```

## مراحل اجرا

### مرحله 1: تعیین محل جاهای پارک

ابتدا باید مشخص کنید هر جای پارک در ویدئوی شما کجاست. برای این کار:

```bash
python spot_selector.py
```

- روی هر جای پارک، **۴ نقطه گوشه** را با کلیک چپ ماوس مشخص کنید.
- بعد از انتخاب ۴ نقطه، کلید `n` را بزنید تا این جای پارک ثبت شود و
  بتوانید جای پارک بعدی را شروع کنید.
- بعد از تعیین تمام جاهای پارک، کلید `s` را بزنید تا مختصات در فایل
  `parking_spots.json` ذخیره شود.
- برای خروج بدون ذخیره، کلید `q` را بزنید.

### مرحله 2: اجرای سیستم اصلی

```bash
python main.py
```

برنامه ویدئوی ورودی (`parking_video.mp4` یا وبکم با تغییر `VIDEO_SOURCE`
به `0`) را پردازش می‌کند و:

- وضعیت هر جای پارک را با رنگ سبز (خالی) یا قرمز (اشغال) نشان می‌دهد.
- ورود/خروج خودروها را از خط مجازی زرد رنگ تشخیص می‌دهد.
- یک داشبورد آماری در گوشه بالا-سمت‌چپ تصویر نمایش می‌دهد.
- ویدئوی نهایی را در `parking_output.avi` ذخیره می‌کند.
- رویدادها را در دو فایل `spot_events.csv` و `vehicle_events.csv`
  با timestamp ثبت می‌کند.

برای خروج از برنامه، کلید `q` را فشار دهید.

## تنظیمات قابل تغییر

در فایل `main.py`:
- `VIDEO_SOURCE`: مسیر ویدئو یا `0` برای وبکم
- `SPOTS_JSON_PATH`: مسیر فایل مختصات جاهای پارک
- `OUTPUT_VIDEO_PATH`: مسیر ذخیره ویدئوی خروجی

در فایل `detector.py` (هنگام ساخت `ParkingDetector`):
- `motion_threshold_ratio`: حساسیت تشخیص اشغال (پیش‌فرض 0.15)
- `vote_threshold`: تعداد فریم لازم برای تأیید تغییر وضعیت (پیش‌فرض 8،
  هرچه بیشتر باشد flickering کمتر می‌شود ولی واکنش سیستم کندتر می‌شود)

در فایل `tracker.py` (هنگام ساخت `VehicleTracker`):
- `line_y_ratio`: موقعیت خط مجازی ورود/خروج (پیش‌فرض 0.5 یعنی وسط تصویر)

## فایل‌های خروجی

| فایل | توضیح |
|---|---|
| `parking_output.avi` | ویدئوی نهایی پردازش‌شده با تمام overlay ها |
| `spot_events.csv` | تاریخچه تغییر وضعیت هر جای پارک (پارک شدن/ترک کردن) |
| `vehicle_events.csv` | تاریخچه ورود/خروج خودروها از خط مجازی |
| `parking_spots.json` | مختصات جاهای پارک تعیین‌شده |
